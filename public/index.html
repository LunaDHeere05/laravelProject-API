Nieuwe gebruiker aanmaken

Dit endpoint voldoet aan de volgende functionele vereisten:

  • Een nieuwe instantie van de entiteit toevoegen aan de database.

POST /users

Body parameters:

  • name (required)
  • email (required)
  • password (required)
  • date_birth
  • abt_me
  • picture
  • is_admin

voorbeeld parameters:

            
                {
                    "name": "joske",
                    "email": "joske.desmet@student.ehb.be",
                    "password": "Tester123!",
                    "abt_me": "hallotjes! Ik ben joske."
                }

Response:

            
                {
                    "message": "User  created successfully!",
                    "userId": 7
                }

de lijnen code waar u de logica voor kan vinden:

  • userModel.js: lijn 121-153
  • index.js: lijn 62-78
  • validation.js: lijn 45-115

Gebruiker bijwerken

Dit endpoint voldoet aan de volgende functionele vereisten:

  • Een bestaande instantie van de entiteit updaten.

PUT /users/:id

Query parameters:

  • id: het ID van de gebruiker die u wilt bijwerken (verplicht).

Body parameters:

  • name
  • email
  • password
  • date_birth
  • abt_me
  • is_admin
  • picture

voorbeeld parameters:

            
                {
                    "name": "josianne"
                }

Response:

            
                {
                    "message": "User  updated successfully"
                }

de lijnen code waar u de logica voor kan vinden:

  • userModel.js: lijn 52-104
  • index.js: lijn 80-100
  • validation.js: lijn 45-115

GET /posts
